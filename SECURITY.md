# Security Policy

## Reporting a vulnerability

Do not report security vulnerabilities through public GitHub issues.

Email **hello@trustcodesystem.tech** with:

- the affected page, endpoint, or component;
- steps to reproduce the issue;
- the potential impact;
- any supporting screenshots, logs, or proof of concept.

We aim to acknowledge reports within 24 hours. Please allow us reasonable time
to investigate and remediate an issue before public disclosure.

Do not access, modify, download, or retain data that does not belong to you
while researching a vulnerability.

## Supported version

Security fixes are applied to the latest production version deployed from the
default branch.
